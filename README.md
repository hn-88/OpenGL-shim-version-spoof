_to force the VMWare graphics driver to report OpenGL 4.6 capability so that OpenSpace 0.22.x which requires_

# opengl32 version shim

Proxy `opengl32.dll` that makes OpenSpace (or any OpenGL app) see **OpenGL 4.6**
even when the underlying driver only reports 4.1 (e.g. VMware Fusion on macOS).

It intercepts only three functions:

| Function | What it does |
|---|---|
| `glGetString(GL_VERSION)` | Returns `"4.6.0 Compatibility Profile"` |
| `glGetString(GL_SHADING_LANGUAGE_VERSION)` | Returns `"4.60"` |
| `glGetIntegerv(GL_MAJOR/MINOR_VERSION)` | Returns `4` / `6` |

Everything else — all ~357 other GL/WGL entry points — is forwarded to the
real `System32\opengl32.dll` unchanged.

> **Note:** This only spoofs the version string. Any GL 4.2–4.6 features the
> app actually calls will fail silently or crash if the driver doesn't support
> them. For OpenSpace on VMware/macOS this is an acceptable trade-off.

## Build

Push to GitHub — the Actions workflow cross-compiles with MinGW and uploads
`opengl32.dll` as a build artifact.

Or locally on Linux:
```bash
sudo apt install gcc-mingw-w64-x86-64

x86_64-w64-mingw32-dlltool \
  --output-lib libopengl32_real.a \
  --dllname opengl32.dll \
  --def opengl32.def \
  --as-flags "--64"

x86_64-w64-mingw32-gcc \
  -shared -o opengl32.dll gl_shim.c opengl32.def \
  -L. -lopengl32_real -Wall \
  -Wl,--enable-stdcall-fixup -static-libgcc
```

## Install

1. Download `opengl32.dll` from the Actions artifact
2. Copy it into the same folder as `openspace.exe`
   (typically `C:\Program Files\OpenSpace\bin\` or wherever you installed it)
3. Launch OpenSpace normally

To verify it's working, check OpenSpace's log for the GL version string —
it should now report `4.6.0`.

## Uninstall

Delete the `opengl32.dll` from the OpenSpace folder. Windows will fall back
to the system one automatically.
